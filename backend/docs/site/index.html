Base de datos

Modo actual configurable en backend/.env con DEMO_MODE=true (archivo local) o Supabase PostgreSQL.

  • Demo local: backend/data/demo-db.json
  • Supabase: ejecutar supabase/schema.sql, schema-admin.sql, schema-ubicacion.sql

Modelo de datos

items          → herramienta (nombre, marca, modelo, tipo, detalle, activo)
contenedores   → ubicación (armario + estante + contenedor opcional)
stock          → cantidad por ítem en una ubicación
movimientos    → egreso / ingreso (historial)

Ubicación física

CódigoSignificado
A00Armario Papelería
A01Armario Herramientas
A02Armario Electrónica
E01–E09Estante (obligatorio)
C01–C99Caja tipo C (opcional)
B00–B99Caja tipo B (opcional)
H01–H99Caja tipo H (opcional)
SCSin contenedor (explícito en QR/código)

Código generado: A01-E03 o A01-E03-C05
